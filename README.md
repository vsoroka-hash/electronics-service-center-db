# Electronics Service Center DB

База даних і прикладний застосунок для курсового проєкту на тему розробки інформаційної системи сервісного центру ремонту електроніки.

## Призначення

Проєкт автоматизує облік:

- клієнтів;
- пристроїв;
- ремонтних замовлень;
- діагностики;
- послуг;
- запчастин;
- платежів;
- аналітичних звітів.

## Склад проєкту

- `schema.sql` — структура MySQL-бази даних, початкові дані, `VIEW`, процедури, тригери.
- `queries.sql` — приклади SQL-запитів для перевірки й демонстрації роботи БД.
- `app.py` — точка входу в програму.
- `reporting.py` — генерація діаграм і графіків.
- `service_center/` — основний Python-код застосунку.
- `diagrams/` — вихідні діаграми у форматі `PlantUML`.
- `tools/plantuml.jar` — локальний `PlantUML` для рендеру PNG.

## Основні можливості

- ведення довідників клієнтів, пристроїв, послуг і запчастин;
- створення та супровід ремонтних замовлень;
- додавання діагностики, послуг, запчастин і платежів;
- контроль суми замовлення та складських залишків на рівні БД;
- формування звітів за доходом, статусами й навантаженням майстрів;
- побудова ER-діаграми та логічної схеми БД.

## Технології

- Python 3.13+
- MySQL 8.x
- Tkinter
- `mysql-connector-python`
- `matplotlib`
- `PlantUML`

## Встановлення залежностей

```bash
pip install -r requirements.txt
```

## Параметри підключення до БД

За замовчуванням використовуються такі змінні середовища:

- `DB_HOST=127.0.0.1`
- `DB_PORT=3306`
- `DB_USER=course_user`
- `DB_PASSWORD=1910`
- `DB_NAME=service_center_db`

За потреби їх можна перевизначити вручну.

## Ініціалізація бази даних

Створення або повне перевідновлення схеми:

```bash
mysql --default-character-set=utf8mb4 -u root -p < schema.sql
```

Запуск демонстраційних SQL-запитів:

```bash
mysql --default-character-set=utf8mb4 -u course_user -p service_center_db < queries.sql
```

## Запуск застосунку

Графічний інтерфейс:

```bash
python app.py
```

Консольний інтерфейс:

```bash
python app.py --cli
```

## Генерація діаграм і графіків

```bash
python reporting.py
```

Після запуску буде створено папку `artifacts/` з файлами:

- `er_diagram.png`
- `schema_diagram.png`
- `revenue_chart.png`
- `status_chart.png`
- `technician_chart.png`

## Архітектура

- `service_center/config.py` — конфігурація з’єднання з БД.
- `service_center/database.py` — доступ до MySQL і пул з’єднань.
- `service_center/services.py` — бізнес-логіка.
- `service_center/gui.py` — графічний інтерфейс.
- `service_center/cli.py` — консольний інтерфейс.
- `service_center/visualization.py` — побудова діаграм і графіків.
- `service_center/table_view.py` — табличне форматування для CLI.

## Перевірка працездатності

Мінімальний сценарій перевірки:

1. Імпортувати `schema.sql` у MySQL.
2. Запустити `python app.py`.
3. Перевірити перегляд замовлень, клієнтів і довідників.
4. Запустити `python reporting.py`.
5. Переконатися, що в `artifacts/` з’явились PNG-файли.

## Примітка

Папки `artifacts/` і `__pycache__/` не зберігаються в репозиторії, оскільки це згенеровані файли.
