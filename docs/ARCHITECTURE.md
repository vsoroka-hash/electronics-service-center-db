# Architecture Overview

## Загальна структура

Проєкт побудований як невеликий шаровий застосунок:

- `app.py` — точка входу;
- `service_center/gui.py` — графічний інтерфейс;
- `service_center/cli.py` — консольний інтерфейс;
- `service_center/services.py` — бізнес-логіка;
- `service_center/database.py` — доступ до MySQL;
- `service_center/visualization.py` — генерація діаграм і графіків.

## Рівні відповідальності

### Presentation layer

- GUI на `Tkinter`;
- CLI для резервного сценарію роботи;
- відображення таблиць, форм і статусів виконання.

### Service layer

- інкапсулює операції з клієнтами, пристроями, замовленнями, платежами та звітами;
- не містить коду інтерфейсу.

### Data access layer

- пул з’єднань із MySQL;
- виконання SQL-запитів;
- виклики збережених процедур.

### Visualization layer

- аналітичні графіки на `matplotlib`;
- структурні діаграми через `PlantUML`.

## База даних

Логіка цілісності частково перенесена в MySQL:

- зовнішні ключі;
- `CHECK`-обмеження;
- `VIEW`;
- процедури;
- тригери.

Такий підхід зменшує дублювання бізнес-правил у Python-коді.
